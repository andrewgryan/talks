Class 4 framework

  • Same observations for all models
  • Observations grouped into 24hr window
  • Forecasts valid at noon on same day
  • Forecasts mapped to observation space

    • No temporal interpolation

Map to observation space

  • Ocean model interpolator
    • Large Fortran code base
    • Complicated build system
    • Monolithic design
    • Fortran namelists
    • Third party release cycle
    • Many required input files
    • Libraries only available on super computer
    • Designed for different purpose
  • Re-written Python package
    • Initially labour intensive
    • Simple API
    • Numpy array inputs
    • No need for input files
    • No Fortran namelist manipulation
    • No need for supercomputer scheduler
    • Optimised search algorithms
